BQ25756RRVR: MCU Control Reference
Project: KVM
Date: 2026-07-15

Purpose
-------

In this project, BQ25756 is used only in reverse mode:

    VBUS_C0 (common 5...48 V input bus, SRP/SRN side)
        -> BQ25756
        -> corresponding USB-C port output (ACP/ACN/VAC side)

The MCU controls each BQ25756 directly over a separate I2C bus. Every BQ25756
uses the fixed 7-bit address 0x6B, so two BQ devices cannot share one physical
I2C bus unless an I2C multiplexer is added.

The MCU performs the following functions:

1. Before a consumer is connected, keep the port output disabled or prepare
   the mandatory 5 V level required to begin USB-PD negotiation.
2. Receive connection status and the negotiated PDO/RDO from the PD controller.
3. Calculate the allowed voltage and current from the total available power.
4. Program voltage and current in BQ25756.
5. Enable reverse mode.
6. Verify that the converter has started and periodically read voltage, current,
   and fault status.
7. Disable reverse mode when the device is disconnected or a fault occurs.


0. Whole-board power-on sequence
--------------------------------

Definitions:

    VBUS_C0_R   - raw VBUS directly from USB-C0, before EF0; this net is also
                  connected to the VBUS input of TPD4S480;
    VBUS_C0     - main power bus after EF0;
    +3V3_MCU    - always-on supply for the MCU and PD0 VIN_3V3, generated from
                  VBUS_C0_R;
    +3V3        - main switchable 3.3 V rail;
    EF0         - TPS26630 connecting VBUS_C0_R to VBUS_C0.

PD0 must be strapped into SafeMode:

    ADCIN1 -> LDO_3V3   (decoded value 7)
    ADCIN2 -> GND       (decoded value 0)

EF0 is enabled only when both devices allow it:

    EF0_ON = PD0_POWER_PATH_ALLOW AND MCU_EF0_ALLOW

In the schematic, the MCU signal is active-high inhibit and is named
EF0_HOLD_OFF:

    EF0_HOLD_OFF = HIGH or Hi-Z -> MCU prevents EF0 from turning on;
    EF0_HOLD_OFF = LOW          -> MCU allows EF0 to turn on.

Actual startup sequence:

1. A source is connected to USB-C0. The initial 5 V appears on VBUS_C0_R.
2. LMR51610Y generates +3V3_MCU directly from VBUS_C0_R.
3. MCU and PD0 boot. PD0 remains in SafeMode and EF0 remains off.
4. During reset, the MCU GPIO is Hi-Z. A pull-up keeps EF0_HOLD_OFF HIGH, so EF0
   cannot turn on physically.
5. The MCU loads the patch bundle and Application Configuration into PD0 over
   I2Ct.
6. PD0 enters APP mode and begins USB-PD negotiation with the source.
7. The MCU waits until PD0 reports an active contract after PS_RDY, then reads
   the active PDO/RDO and the negotiated voltage and current.
8. The MCU validates the profile: allowed voltage, current, power, and absence
   of PD0 faults. EF0 remains off until validation is complete.
9. If the profile is valid, the MCU configures EF0_HOLD_OFF as OUTPUT LOW.
10. EF0 turns on only if PD0 simultaneously asserts POWER_PATH_EN.
11. TPS26630 performs soft-start, after which the negotiated input voltage
    appears on VBUS_C0.
12. The MCU waits for PGOOD_EF0 = HIGH and checks FLT#_EF0.
13. After PGOOD, the MCU programs and enables the +5 V BQ25756 channel, then
    enables the main +3V3 rail through EN_3V3.
14. The remaining PD controllers and output ports may start only after +5 V and
    +3V3 are stable.

On detach, Hard Reset, loss of contract, or PD0 fault, PD0 deasserts
POWER_PATH_EN and turns EF0 off independently of the MCU. After receiving the
corresponding IRQ, the MCU must:

1. Disable the output BQ25756 converters and the main +3V3 rail.
2. Return EF0_HOLD_OFF to HIGH or Hi-Z.
3. Wait for a new valid contract.
4. Repeat profile validation and the EF0 enable sequence.


1. I2C electrical interface
---------------------------

7-bit BQ25756 address: 0x6B

If the MCU driver expects an already shifted 8-bit address:

    write: 0xD6
    read:  0xD7

Most modern HALs accept the 7-bit address 0x6B. Do not shift the address again
if the HAL already does so.

Supported speeds:

    Standard Mode:       up to 100 kbit/s
    Fast Mode:           up to 400 kbit/s
    Fast Mode Plus:      up to 1 Mbit/s

The recommended project setting is 400 kbit/s.

SDA and SCL are open-drain. External pull-ups to 3.3 V must remain in the
schematic.


2. Transaction format
---------------------

Writing an 8-bit register:

    START
    0x6B + WRITE
    register address
    value
    STOP

Reading an 8-bit register:

    START
    0x6B + WRITE
    register address
    REPEATED START
    0x6B + READ
    value
    STOP

16-bit registers are little-endian:

    the low byte is stored at the lower address;
    the high byte is stored at the next address.

Example: writing 0x03E8 to REG0x0C:

    address 0x0C <- 0xE8
    address 0x0D <- 0x03

Write both bytes of a 16-bit register in one sequential multi-write transaction.
Do not write only one byte of a 16-bit register.

Required MCU driver functions:

    bq_write8(bus, register, value)
    bq_read8(bus, register)
    bq_write16_le(bus, register, value)
    bq_read16_le(bus, register)
    bq_update_bits(bus, register, mask, value)

bq_update_bits() must first read the register, change only the selected bits,
and write the result back. This prevents accidental modification of reserved
bits.


3. Registers required by the project
------------------------------------

REG0x0A/0x0B - IAC_REV, reverse-mode output-current limit
--------------------------------------------------------------------------

16-bit, little-endian.

Purpose: controls current on the VAC/ACP/ACN side, i.e. current delivered to the
USB-C port.

For the installed RAC_SNS = 5 mOhm:

    range: 0.4...20 A
    step:  50 mA
    field: bits [10:2]

Calculation:

    code = I_limit_mA / 50
    REG0x0A = code << 2

Examples:

    0.5 A -> 0x0028    bytes: 28 00
    1.0 A -> 0x0050    bytes: 50 00
    1.5 A -> 0x0078    bytes: 78 00
    2.0 A -> 0x00A0    bytes: A0 00
    3.0 A -> 0x00F0    bytes: F0 00
    5.0 A -> 0x0190    bytes: 90 01

The actual limit cannot exceed the hardware limit set by the ILIM_HIZ resistor.
With RILIM = 3.6 kOhm and RAC_SNS = 5 mOhm, the hardware limit is approximately
5.56 A.


REG0x0C/0x0D - VAC_REV, reverse-mode output voltage
--------------------------------------------------------------------------

16-bit, little-endian.

Purpose: sets the voltage at VAC, i.e. the converter output.

    range: 3.3...65 V
    step:  20 mV
    field: bits [13:2]

Calculation:

    code = V_out_mV / 20
    REG0x0C = code << 2

Equivalent shortened formula:

    REG0x0C = V_out_mV / 5

The value must be limited to the project's allowed range of 5...20 V.

Examples:

     5 V -> 0x03E8    bytes: E8 03
     9 V -> 0x0708    bytes: 08 07
    12 V -> 0x0960    bytes: 60 09
    15 V -> 0x0BB8    bytes: B8 0B
    20 V -> 0x0FA0    bytes: A0 0F


REG0x15 - WATCHDOG
--------------------------------------------------------------------------

8-bit. WATCHDOG field = bits [5:4].

    00b = disabled
    01b = 40 seconds
    10b = 80 seconds
    11b = 160 seconds

Recommended for the first firmware revision: disable the watchdog:

    bq_update_bits(0x15, 0x30, 0x00)

Reason: when the watchdog expires, EN_REV and some other settings are reset. If
the watchdog is enabled later, the MCU must write 1 to WD_RST, REG0x17 bit 5,
before the selected period expires.


REG0x17 - forward-mode disable and watchdog service
--------------------------------------------------------------------------

Required bits:

    bit 5 WD_RST   - write 1 to reset the watchdog; automatically returns to 0
    bit 2 EN_HIZ   - must remain 0 for converter operation
    bit 0 EN_CHG   - forward charging; must be 0 in this project

Initialization:

    bq_update_bits(0x17, 0x05, 0x00)

This command clears EN_HIZ and EN_CHG while preserving all other bits.

If the watchdog is used:

    bq_update_bits(0x17, 0x20, 0x20)


REG0x18 - hardware current-pin control
--------------------------------------------------------------------------

Required bit:

    bit 6 EN_ILIM_HIZ_PIN
        1 = use the ILIM_HIZ hardware limit
        0 = ignore the hardware limit

This bit must remain 1 because the 3.6 kOhm resistor provides an independent
hardware limit of approximately 5.56 A.

Verification command:

    bq_update_bits(0x18, 0x40, 0x40)


REG0x19 - reverse-mode enable and disable
--------------------------------------------------------------------------

Required bits:

    bit 7 REG_RST     - reset registers to default values
    bit 6 EN_IAC_LOAD - internal load on the VAC side; normally 0
    bit 5 EN_PFM      - PFM at light load
    bit 0 EN_REV      - reverse-mode control

Enable output:

    bq_update_bits(0x19, 0x01, 0x01)

Disable output:

    bq_update_bits(0x19, 0x01, 0x00)

Reset all programmable registers:

    bq_update_bits(0x19, 0x80, 0x80)

Use REG_RST only during initialization or fault recovery. All settings must be
written again after REG_RST. Bit 7 automatically returns to 0.


REG0x1C - TS control
--------------------------------------------------------------------------

    bit 0 EN_TS
        1 = monitor TS in forward and reverse modes
        0 = ignore TS

In the current schematic, TS is held in the normal range by a 10 kOhm / 10 kOhm
divider, so EN_TS may remain 1. If the divider is removed, install a real NTC or
clear EN_TS before enabling reverse mode.


REG0x1E - output undervoltage shutdown threshold
--------------------------------------------------------------------------

    bit 5 SYSREV_UV
        0 = shut down when VAC falls below 80% of programmed VAC_REV
        1 = fixed 3.3 V threshold

Recommended setting: 0.

    bq_update_bits(0x1E, 0x20, 0x00)

This prevents a 20 V output from continuing to operate after a severe collapse,
for example down to 4 V. The IC clears EN_REV automatically when REV_UVP trips.


REG0x21 - primary status
--------------------------------------------------------------------------

Read-only.

    bit 7 ADC_DONE_STAT  - one-shot ADC complete
    bit 6 IAC_DPM_STAT   - input/output current limit reached
    bit 5 VAC_DPM_STAT   - converter regulating at voltage limit
    bit 3 WD_STAT        - watchdog expired

WD_STAT must be 0 during normal operation. IAC_DPM_STAT may assert briefly at
maximum load, but a persistent state means the consumer has reached the
programmed current limit.


REG0x22 - temperature and power status
--------------------------------------------------------------------------

Read-only.

    bit 7 PG_STAT
    bits [6:4] TS_STAT

TS_STAT:

    000 = Normal
    001 = Warm
    010 = Cool
    011 = Cold
    100 = Hot

If EN_TS = 1, TS_STAT must be Normal before reverse mode can start.


REG0x23 - reverse-mode confirmation
--------------------------------------------------------------------------

Read-only.

    bit 2 REVERSE_STAT
        0 = reverse mode disabled
        1 = reverse mode operating

After setting EN_REV, the MCU must wait for REVERSE_STAT = 1. The datasheet does
not specify a guaranteed software wait time. Firmware must use a finite timeout,
for example 100 ms as a project value, rather than an infinite loop.


REG0x24 - current fault states
--------------------------------------------------------------------------

Read-only.

    bit 7 VAC_UV_STAT    - VAC below allowed level
    bit 6 VAC_OV_STAT    - VAC above allowed level
    bit 5 IBAT_OCP_STAT  - overcurrent on the SRP/SRN side
    bit 4 VBAT_OV_STAT   - overvoltage on the SRP/SRN side
    bit 3 TSHUT_STAT     - thermal shutdown
    bit 2 CHG_TMR_STAT   - forward-charge timer; unused in this project
    bit 1 DRV_OKZ_STAT   - DRV_SUP outside the allowed range

Project-critical bits: 7, 6, 5, 4, 3, and 1.


REG0x25, REG0x26, REG0x27 - event flags
--------------------------------------------------------------------------

Flags are ClearOnRead. Even if INT is not connected, the MCU can poll these
registers for diagnostics.

REG0x25:

    bit 7 ADC_DONE_FLAG
    bit 6 IAC_DPM_FLAG
    bit 5 VAC_DPM_FLAG
    bit 3 WD_FLAG

REG0x26:

    bit 7 PG_FLAG
    bit 4 TS_FLAG
    bit 3 REVERSE_FLAG
    bit 1 FSW_SYNC_FLAG

REG0x27:

    bit 7 VAC_UV_FLAG
    bit 6 VAC_OV_FLAG
    bit 5 IBAT_OCP_FLAG
    bit 4 VBAT_OV_FLAG
    bit 3 TSHUT_FLAG
    bit 2 CHG_TMR_FLAG
    bit 1 DRV_OKZ_FLAG


REG0x2B - ADC control
--------------------------------------------------------------------------

    bit 7 ADC_EN
    bit 6 ADC_RATE: 0 = continuous, 1 = one-shot
    bits [5:4] ADC_SAMPLE:
        00 = 15 bit
        01 = 14 bit
        10 = 13 bit
    bit 3 ADC_AVG
    bit 2 ADC_AVG_INIT

Recommended simple continuous-monitoring mode, 13 bit:

    bq_write8(0x2B, 0xA0)

Stop ADC:

    bq_update_bits(0x2B, 0x80, 0x00)

One-shot, 13-bit mode:

    bq_write8(0x2B, 0xE0)

After a one-shot conversion, the MCU waits for REG0x21 bit 7 = 1. ADC_EN clears
automatically when the conversion is complete.


REG0x2C - ADC channel selection
--------------------------------------------------------------------------

In this register, 1 disables the corresponding channel:

    bit 7 IAC_ADC_DIS
    bit 6 IBAT_ADC_DIS
    bit 5 VAC_ADC_DIS
    bit 4 VBAT_ADC_DIS
    bit 2 TS_ADC_DIS
    bit 1 VFB_ADC_DIS

The project requires IAC, IBAT, VAC, and VBAT. The default value already leaves
these channels enabled. Avoid rewriting REG0x2C unless necessary.


REG0x2D/0x2E - measured IAC current
--------------------------------------------------------------------------

16-bit, little-endian, signed two's complement.

For RAC_SNS = 5 mOhm:

    IAC_mA = signed16(raw) * 0.8

This is the current on the USB-C output side.


REG0x2F/0x30 - measured IBAT current
--------------------------------------------------------------------------

16-bit, little-endian, signed two's complement.

For RBAT_SNS = 5 mOhm:

    IBAT_mA = signed16(raw) * 2

This is the current drawn by the converter from the common VBUS_C0 bus.


REG0x31/0x32 - measured VAC voltage
--------------------------------------------------------------------------

16-bit, little-endian, unsigned.

    VAC_mV = raw * 2

This is the actual converter output voltage.


REG0x33/0x34 - measured VBAT voltage
--------------------------------------------------------------------------

16-bit, little-endian, unsigned.

    VBAT_mV = raw * 2

In this project, this is the actual common VBUS_C0 voltage on the SRN side.


REG0x3B and REG0x3C - gate-drive speed and dead time
--------------------------------------------------------------------------

The defaults select the fastest gate drivers and 45 ns dead time. Keep the
default values during initial bring-up. Change them only after checking SW1/SW2
ringing, shoot-through, and edge rates with an oscilloscope.


REG0x3D - device identification
--------------------------------------------------------------------------

Read-only. Expected value for the current datasheet revision: 0x12.

    bits [6:3] PART_NUM = 010b, BQ25756
    bits [2:0] DEV_REV  = 010b

If a different value is read, the MCU must not enable the power stage until the
installed revision has been confirmed to be compatible.


REG0x62 - reverse-mode input current and dynamic compensation
--------------------------------------------------------------------------

    bits [7:6] IBAT_REV:
        00 = 20 A
        01 = 15 A
        10 = 10 A
        11 = 5 A

    bit 1 EN_CONV_FAST_TRANSIENT:
        0 = normal compensation
        1 = fast reverse-mode compensation

The project has approximately 200 uF at the output. Clear
EN_CONV_FAST_TRANSIENT before setting EN_REV:

    bq_update_bits(0x62, 0x02, 0x00)

The datasheet requires substantially more capacitance in reverse boost mode when
this bit is 1.

IBAT_REV limits the current drawn by one converter from VBUS_C0. Estimate the
required current as:

    I_VBUS_C0_expected = Vout * Iout / (VBUS_C0 * efficiency)

Select the next higher 5/10/15/20 A limit. Do not select 5 A for 5 V -> 5 V at
5 A: losses make the input current greater than 5 A. That operating point
requires the 10 A limit or a reduced permitted output current.

Example: select 10 A and EN_CONV_FAST_TRANSIENT = 0:

    bq_update_bits(0x62, 0xC2, 0x80)


4. Conditions required for EN_REV to work
-----------------------------------------

BQ25756 enables reverse mode only when all of the following are true:

1. SRN is above 3 V.
2. DRV_SUP is within its allowed range.
3. EN_REV is written to 1.
4. TS is between BCOLD and BHOT when EN_TS = 1.

If REVERSE_STAT does not become 1, the MCU must read REG0x22, REG0x23, and
REG0x24 instead of retrying EN_REV indefinitely.


5. Complete startup sequence for one BQ25756
---------------------------------------------

Step 1. Confirm that the common input bus is present

    VBUS_C0 > 3 V
    external +5V_DRV for DRV_SUP is present

Step 2. Verify communication

    id = bq_read8(0x3D)
    expect id = 0x12

Step 3. Keep reverse mode disabled

    bq_update_bits(0x19, 0x01, 0x00)

Step 4. Disable the watchdog

    bq_update_bits(0x15, 0x30, 0x00)

Step 5. Disable forward charging and HIZ

    bq_update_bits(0x17, 0x05, 0x00)

Step 6. Keep ILIM_HIZ hardware protection enabled

    bq_update_bits(0x18, 0x40, 0x40)

Step 7. Select the normal reverse-undervoltage threshold

    bq_update_bits(0x1E, 0x20, 0x00)

Step 8. Disable fast transient compensation

    bq_update_bits(0x62, 0x02, 0x00)

Step 9. Set IBAT_REV from the available input power

    bq_update_bits(0x62, 0xC0, selected_value)

Step 10. Set a safe initial output current

    bq_write16_le(0x0A, IAC_REV_value)

Step 11. Set the required output voltage

    bq_write16_le(0x0C, VAC_REV_value)

Step 12. Read current faults

    fault = bq_read8(0x24)
    if fault & 0x7A != 0: do not enable

Mask 0x7A checks VAC_OV, IBAT_OCP, VBAT_OV, TSHUT, and DRV_OKZ. At startup,
VAC_UV may simply indicate that the output is off, so evaluate it separately
according to the port state.

Step 13. Enable reverse mode

    bq_update_bits(0x19, 0x01, 0x01)

Step 14. Wait for confirmation

    wait until bq_read8(0x23) & 0x04 != 0
    use a finite timeout

Step 15. Verify the actual voltage

    VAC = bq_read16_le(0x31) * 2 mV

Allow the PD controller to complete the power transition only after VAC reaches
the target value within the selected firmware tolerance.

Step 16. Start ADC and periodic monitoring

    bq_write8(0x2B, 0xA0)


6. Operation with a USB-PD controller
-------------------------------------

Initial state of a USB-C source port:

1. The MCU knows the total power available from the input PD port.
2. The MCU calculates the set of Source PDOs allowed for each output port.
3. Before negotiation, the output BQ25756 is prepared for 5 V.
4. The PD controller advertises the Source PDOs allowed by the MCU.
5. The consumer selects a PDO and sends a Request/RDO.
6. The PD controller reports the selected voltage and current to the MCU.
7. The MCU programs IAC_REV first, then VAC_REV in the corresponding BQ25756.
8. The MCU checks REVERSE_STAT and measured VAC.
9. Only after the voltage is reached may the PD controller continue the
   transition to the new contract and report source readiness to the consumer.

The fixed 5 V channel uses the same sequence and I2C driver. Its only difference
is the constant VAC_REV value 0x03E8.

When the active contract changes:

1. Never program a voltage higher than the PDO selected by the consumer.
2. When reducing permitted current, reduce IAC_REV first.
3. When changing voltage, write the new VAC_REV and monitor VAC_ADC.
4. After stabilization, set the final IAC_REV if it was temporarily reduced.

BQ25756 does not know the total limit of the input PD contract. The MCU must
enforce it:

    sum(Vout_i * Iout_limit_i / efficiency_i) <= available input power

Reserve margin for the MCU, PD controllers, conversion losses, and transients.


7. Periodic monitoring
----------------------

Recommended firmware polling interval: 10...50 ms under load and 100...500 ms
while idle. These are project values, not datasheet requirements.

Read on every cycle:

    REG0x23  REVERSE_STAT
    REG0x24  current faults
    REG0x2D  IAC
    REG0x2F  IBAT
    REG0x31  VAC
    REG0x33  VBAT/VBUS_C0

Periodically read the ClearOnRead flags for logging:

    REG0x25
    REG0x26
    REG0x27

The MCU must also calculate:

    Pout = VAC * IAC
    Pin_channel = VBAT * IBAT

If total channel consumption approaches the available input power, the MCU must
reduce IAC_REV limits and update the advertised PD profiles.


8. Fault handling
-----------------

I2C lost or PART_NUM incorrect:

    Do not enable EN_REV. Disable the external port power path, if present.

REVERSE_STAT unexpectedly becomes 0:

    read REG0x24 and REG0x27;
    treat the power contract as violated;
    notify the PD controller or disable the port;
    do not restart until the cause is known.

VAC_UV or REV_UVP:

    BQ25756 automatically clears EN_REV;
    check overload, IAC_REV, and available input power;
    after removing the load, repeat the complete startup sequence.

VAC_OV:

    immediately treat the output as faulty;
    clear EN_REV;
    do not resume operation until measured VAC is safe.

IBAT_OCP:

    reduce the power allowed for the port;
    check the selected IBAT_REV and the common input bus.

TSHUT:

    clear EN_REV;
    wait for the converter to cool;
    reduce current or prohibit the high-power profile.

DRV_OKZ:

    clear EN_REV;
    check external +5V_DRV;
    allow restart only after REG0x24 bit 1 clears.

TS Cold/Hot:

    BQ25756 clears EN_REV;
    with the present fixed divider, this event indicates a hardware fault in
    the TS or REGN circuitry.


9. Normal shutdown sequence
---------------------------

1. Tell the PD controller to terminate or reset the contract if a device is
   connected.
2. Reduce IAC_REV to the minimum required value if a gradual load reduction is
   required.
3. Clear EN_REV:

       bq_update_bits(0x19, 0x01, 0x00)

4. Wait for REVERSE_STAT = 0.
5. Check VAC_ADC.

Output capacitors may remain charged after EN_REV = 0. A controlled discharge
path is required if VBUS must fall quickly and predictably. Do not treat EN_REV
as a physical output-discharge circuit.


10. Minimal pseudocode
----------------------

bool bq_start_reverse(bus, vout_mV, iout_mA, ibat_limit_bits)
{
    if (bq_read8(bus, 0x3D) != 0x12)
        return false;

    bq_update_bits(bus, 0x19, 0x01, 0x00); // EN_REV = 0
    bq_update_bits(bus, 0x15, 0x30, 0x00); // watchdog off
    bq_update_bits(bus, 0x17, 0x05, 0x00); // EN_HIZ=0, EN_CHG=0
    bq_update_bits(bus, 0x18, 0x40, 0x40); // keep ILIM_HIZ enabled
    bq_update_bits(bus, 0x1E, 0x20, 0x00); // REV_UV = 80% target
    bq_update_bits(bus, 0x62, 0xC2, ibat_limit_bits); // fast transient off

    uint16_t iac_rev = (iout_mA / 50) << 2;
    uint16_t vac_rev = (vout_mV / 20) << 2;

    bq_write16_le(bus, 0x0A, iac_rev);
    bq_write16_le(bus, 0x0C, vac_rev);

    uint8_t fault = bq_read8(bus, 0x24);
    if (fault & 0x7A)
        return false;

    bq_update_bits(bus, 0x19, 0x01, 0x01); // EN_REV = 1

    if (!wait_until_with_timeout(bq_read8(bus, 0x23) & 0x04))
        return false;

    bq_write8(bus, 0x2B, 0xA0); // continuous ADC, 13 bit
    return true;
}

In a real implementation, clamp before calculating:

    vout_mV: 5000...20000
    iout_mA: 400...5000

Round values down to 20 mV / 50 mA steps so that the values permitted by the PD
contract are never exceeded.

12. Scope and limitations
-------------------------

This file describes direct MCU control of BQ25756. The specific TPS65987DDH
registers used by the MCU to read the selected PDO/RDO and control PD-port state
must be documented separately: they use a different I2C protocol described in
SLVUBH2B. Do not mix PD-controller register addresses with BQ25756 register
addresses.
