BQ25756RRVR: управление преобразователями от MCU
Проект: KVM
Дата: 2026-07-15

Назначение
==========

В проекте BQ25756 используется только в reverse mode:

    VBUS_C0 (общая входная шина 5...48 В, сторона SRP/SRN)
        -> BQ25756
        -> выход соответствующего USB-C-порта (сторона ACP/ACN/VAC)

MCU напрямую управляет каждым BQ25756 по отдельной I2C-шине. Все BQ25756
имеют фиксированный 7-битный адрес 0x6B, поэтому два BQ нельзя подключать
к одной физической I2C-шине без мультиплексора.

MCU выполняет следующие функции:

1. До подключения потребителя удерживает выход порта выключенным либо
   подготавливает обязательные 5 В для начала USB-PD переговоров.
2. Получает от PD-контроллера информацию о подключении и согласованном PDO/RDO.
3. Вычисляет допустимые напряжение и ток с учетом общей доступной мощности.
4. Записывает напряжение и ток в BQ25756.
5. Включает reverse mode.
6. Проверяет фактический запуск и периодически читает напряжения, токи и ошибки.
7. При отключении устройства или аварии выключает reverse mode.


0. PWR_ON SEQUENCE всей платы
================================

Обозначения:

    VBUS_C0_R   - сырой VBUS непосредственно от USB-C0, до EF0; эта же сеть
                  подключена к VBUS-входу TPD4S480;
    VBUS_C0     - основная силовая шина после EF0;
    +3V3_MCU    - always-on питание MCU и VIN_3V3 PD0 от VBUS_C0_R;
    +3V3        - основная отключаемая шина 3.3 В;
    EF0         - TPS26630, соединяющий VBUS_C0_R с VBUS_C0.

PD0 должен быть аппаратно установлен в SafeMode:

    ADCIN1 -> LDO_3V3   (decoded value 7)
    ADCIN2 -> GND       (decoded value 0)

EF0 разрешается одновременно двумя устройствами:

    EF0_ON = PD0_POWER_PATH_ALLOW AND MCU_EF0_ALLOW

В схеме сигнал MCU имеет обратное имя EF0_HOLD_OFF:

    EF0_HOLD_OFF = HIGH или Hi-Z -> MCU запрещает EF0;
    EF0_HOLD_OFF = LOW           -> MCU разрешает EF0.

Фактическая последовательность включения:

1. К USB-C0 подключается источник. На VBUS_C0_R появляются начальные 5 В.
2. LMR51610Y формирует +3V3_MCU непосредственно из VBUS_C0_R.
3. MCU и PD0 загружаются. PD0 остается в SafeMode, EF0 закрыт.
4. Во время reset GPIO MCU находится в Hi-Z. Подтяжка держит
   EF0_HOLD_OFF = HIGH, поэтому EF0 физически не может открыться.
5. MCU загружает patch bundle и Application Configuration в PD0 через I2Ct.
6. PD0 переходит в APP mode и начинает USB-PD переговоры с источником.
7. MCU ожидает, пока PD0 сообщит об активном контракте после PS_RDY, затем
   читает у PD0 активные PDO/RDO, согласованные напряжение и ток.
8. MCU проверяет профиль: допустимое напряжение, ток, мощность и отсутствие
   ошибок PD0. До завершения проверки EF0 остается закрытым.
9. Если профиль допустим, MCU переводит EF0_HOLD_OFF в OUTPUT LOW.
10. EF0 откроется только при одновременном разрешении POWER_PATH_EN от PD0.
11. TPS26630 выполняет soft-start, после чего на VBUS_C0 появляется
    согласованное входное напряжение.
12. MCU ждет PGOOD_EF0 = HIGH и проверяет FLT#_EF0.
13. После PGOOD MCU программирует и включает BQ25756 канала +5V, затем включает
    основную шину +3V3 сигналом EN_3V3.
14. Только после стабилизации +5V и +3V3 разрешается запуск остальных
    PD-контроллеров и выходных портов.

При detach, Hard Reset, потере контракта или ошибке PD0 снимает
POWER_PATH_EN и закрывает EF0 независимо от MCU. MCU после получения IRQ должен:

1. Выключить выходные BQ25756 и основную +3V3.
2. Вернуть EF0_HOLD_OFF в HIGH или Hi-Z.
3. Дождаться нового допустимого контракта.
4. Повторить проверку профиля и последовательность открытия EF0.


1. Электрический интерфейс I2C
================================

7-битный адрес BQ25756: 0x6B

Если драйвер MCU принимает уже сдвинутый 8-битный адрес:

    запись: 0xD6
    чтение: 0xD7

Большинство современных HAL принимает 7-битный адрес 0x6B. Нельзя дополнительно
сдвигать адрес, если это уже делает HAL.

Поддерживаемая скорость:

    Standard Mode:       до 100 кбит/с
    Fast Mode:           до 400 кбит/с
    Fast Mode Plus:      до 1 Мбит/с

Для проекта рекомендуется 400 кбит/с.

SDA и SCL являются open-drain. В схеме должны оставаться внешние подтяжки к 3.3 В.


2. Формат операций
===================

Запись 8-битного регистра:

    START
    0x6B + WRITE
    адрес регистра
    значение
    STOP

Чтение 8-битного регистра:

    START
    0x6B + WRITE
    адрес регистра
    REPEATED START
    0x6B + READ
    значение
    STOP

16-битные регистры имеют формат little-endian:

    младший байт находится по младшему адресу;
    старший байт находится по следующему адресу.

Пример записи значения 0x03E8 в REG0x0C:

    адрес 0x0C <- 0xE8
    адрес 0x0D <- 0x03

Два байта 16-битного регистра следует записывать одной последовательной
multi-write транзакцией. Не записывать только один байт 16-битного регистра.

Нужные функции драйвера MCU:

    bq_write8(bus, register, value)
    bq_read8(bus, register)
    bq_write16_le(bus, register, value)
    bq_read16_le(bus, register)
    bq_update_bits(bus, register, mask, value)

bq_update_bits() обязательно сначала читает регистр, изменяет только указанные
биты и записывает результат обратно. Это не дает случайно изменить reserved bits.


3. Регистры, необходимые проекту
=================================

REG0x0A/0x0B - IAC_REV, ограничение выходного тока в reverse mode
-------------------------------------------------------------------------------

16 бит, little-endian.

Назначение: регулирует ток на стороне VAC/ACP/ACN, то есть ток, отдаваемый
USB-C-порту.

Для установленного RAC_SNS = 5 мОм:

    диапазон: 0.4...20 А
    шаг:      50 мА
    поле:     bits [10:2]

Расчет:

    code = I_limit_mA / 50
    REG0x0A = code << 2

Примеры:

    0.5 А -> 0x0028    байты: 28 00
    1.0 А -> 0x0050    байты: 50 00
    1.5 А -> 0x0078    байты: 78 00
    2.0 А -> 0x00A0    байты: A0 00
    3.0 А -> 0x00F0    байты: F0 00
    5.0 А -> 0x0190    байты: 90 01

Фактический предел будет не выше аппаратного ограничения, заданного резистором
ILIM_HIZ. При RILIM = 3.6 кОм и RAC_SNS = 5 мОм аппаратный предел составляет
примерно 5.56 А.


REG0x0C/0x0D - VAC_REV, выходное напряжение в reverse mode
-------------------------------------------------------------------------------

16 бит, little-endian.

Назначение: задает напряжение на VAC, то есть на выходе преобразователя.

    диапазон: 3.3...65 В
    шаг:      20 мВ
    поле:     bits [13:2]

Расчет:

    code = V_out_mV / 20
    REG0x0C = code << 2

Эквивалентная сокращенная формула:

    REG0x0C = V_out_mV / 5

Значение должно быть ограничено допустимым диапазоном проекта 5...20 В.

Примеры:

     5 В -> 0x03E8    байты: E8 03
     9 В -> 0x0708    байты: 08 07
    12 В -> 0x0960    байты: 60 09
    15 В -> 0x0BB8    байты: B8 0B
    20 В -> 0x0FA0    байты: A0 0F


REG0x15 - WATCHDOG
-------------------------------------------------------------------------------

8 бит. Поле WATCHDOG = bits [5:4].

    00b = отключен
    01b = 40 секунд
    10b = 80 секунд
    11b = 160 секунд

Рекомендуемый для первой версии прошивки вариант: отключить watchdog:

    bq_update_bits(0x15, 0x30, 0x00)

Причина: при истечении watchdog EN_REV и некоторые другие настройки сбрасываются.
Если watchdog будет включен позднее, MCU должен до истечения периода записывать
1 в WD_RST, REG0x17 bit 5.


REG0x17 - запрет прямого режима и обслуживание watchdog
-------------------------------------------------------------------------------

Нужные биты:

    bit 5 WD_RST   - записать 1 для сброса watchdog; автоматически вернется в 0
    bit 2 EN_HIZ   - должен оставаться 0 для работы преобразователя
    bit 0 EN_CHG   - прямой заряд; в этом проекте должен быть 0

Инициализация:

    bq_update_bits(0x17, 0x05, 0x00)

Эта команда очищает EN_HIZ и EN_CHG. Остальные биты сохраняются.

Если watchdog используется:

    bq_update_bits(0x17, 0x20, 0x20)


REG0x18 - управление аппаратными токовыми пинами
-------------------------------------------------------------------------------

Нужный бит:

    bit 6 EN_ILIM_HIZ_PIN
        1 = аппаратный предел ILIM_HIZ используется
        0 = аппаратный предел игнорируется

В проекте этот бит должен оставаться равным 1, потому что резистор 3.6 кОм является
независимой аппаратной защитой приблизительно на 5.56 А.

Проверочная команда:

    bq_update_bits(0x18, 0x40, 0x40)


REG0x19 - включение и выключение reverse mode
-------------------------------------------------------------------------------

Нужные биты:

    bit 7 REG_RST     - сброс регистров к значениям по умолчанию
    bit 6 EN_IAC_LOAD - внутренняя нагрузка со стороны VAC; обычно 0
    bit 5 EN_PFM      - PFM при малой нагрузке
    bit 0 EN_REV      - управление reverse mode

Включение выхода:

    bq_update_bits(0x19, 0x01, 0x01)

Выключение выхода:

    bq_update_bits(0x19, 0x01, 0x00)

Сброс всех программируемых регистров:

    bq_update_bits(0x19, 0x80, 0x80)

REG_RST использовать только при инициализации или восстановлении после ошибки.
После REG_RST необходимо заново записать все настройки. Сам bit 7 автоматически
вернется в 0.


REG0x1C - контроль TS
-------------------------------------------------------------------------------

    bit 0 EN_TS
        1 = проверять TS в прямом и reverse mode
        0 = игнорировать TS

В текущей схеме TS зафиксирован делителем 10 кОм / 10 кОм в нормальном диапазоне,
поэтому EN_TS можно оставить равным 1. Если делитель будет удален, необходимо либо
сделать настоящий NTC, либо очистить EN_TS до запуска reverse mode.


REG0x1E - порог отключения при просадке выходного напряжения
-------------------------------------------------------------------------------

    bit 5 SYSREV_UV
        0 = отключение при падении VAC ниже 80% заданного VAC_REV
        1 = фиксированный порог 3.3 В

Рекомендуется оставить 0:

    bq_update_bits(0x1E, 0x20, 0x00)

Это не позволит выходу 20 В продолжать работу после сильной просадки, например
до 4 В. При срабатывании REV_UVP микросхема сама очищает EN_REV.


REG0x21 - основной статус
-------------------------------------------------------------------------------

Только чтение.

    bit 7 ADC_DONE_STAT  - one-shot ADC завершен
    bit 6 IAC_DPM_STAT   - достигнуто ограничение входного/выходного тока
    bit 5 VAC_DPM_STAT   - преобразователь регулируется по ограничению напряжения
    bit 3 WD_STAT        - watchdog истек

Для нормальной работы watchdog должен быть 0. IAC_DPM_STAT может кратковременно
включаться при предельной нагрузке, но его длительное состояние означает, что
потребитель уперся в установленный предел тока.


REG0x22 - температура и наличие питания
-------------------------------------------------------------------------------

Только чтение.

    bit 7 PG_STAT
    bits [6:4] TS_STAT

TS_STAT:

    000 = Normal
    001 = Warm
    010 = Cool
    011 = Cold
    100 = Hot

Для запуска reverse mode при EN_TS=1 требуется TS_STAT = Normal.


REG0x23 - подтверждение reverse mode
-------------------------------------------------------------------------------

Только чтение.

    bit 2 REVERSE_STAT
        0 = reverse mode выключен
        1 = reverse mode работает

После установки EN_REV MCU обязан дождаться REVERSE_STAT = 1. Datasheet не задает
гарантированное программное время ожидания. В прошивке следует использовать
конечный timeout, например 100 мс как проектное значение, а не бесконечный цикл.


REG0x24 - текущие аварийные состояния
-------------------------------------------------------------------------------

Только чтение.

    bit 7 VAC_UV_STAT    - VAC ниже допустимого уровня
    bit 6 VAC_OV_STAT    - VAC выше допустимого уровня
    bit 5 IBAT_OCP_STAT  - превышен ток на стороне SRP/SRN
    bit 4 VBAT_OV_STAT   - перенапряжение на стороне SRP/SRN
    bit 3 TSHUT_STAT     - тепловое отключение
    bit 2 CHG_TMR_STAT   - таймер прямого заряда; для проекта не используется
    bit 1 DRV_OKZ_STAT   - DRV_SUP вне допустимого диапазона

Критические для проекта биты: 7, 6, 5, 4, 3 и 1.


REG0x25, REG0x26, REG0x27 - флаги событий
-------------------------------------------------------------------------------

Флаги очищаются чтением (ClearOnRead). Даже если INT не подключен, MCU может
периодически читать эти регистры для диагностики.

REG0x25:

    bit 7 ADC_DONE_FLAG
    bit 6 IAC_DPM_FLAG
    bit 5 VAC_DPM_FLAG
    bit 3 WD_FLAG

REG0x26:

    bit 7 PG_FLAG
    bit 4 TS_FLAG
    bit 3 REVERSE_FLAG
    bit 1 FSW_SYNC_FLAG

REG0x27:

    bit 7 VAC_UV_FLAG
    bit 6 VAC_OV_FLAG
    bit 5 IBAT_OCP_FLAG
    bit 4 VBAT_OV_FLAG
    bit 3 TSHUT_FLAG
    bit 2 CHG_TMR_FLAG
    bit 1 DRV_OKZ_FLAG


REG0x2B - управление ADC
-------------------------------------------------------------------------------

    bit 7 ADC_EN
    bit 6 ADC_RATE: 0 = continuous, 1 = one-shot
    bits [5:4] ADC_SAMPLE:
        00 = 15 бит
        01 = 14 бит
        10 = 13 бит
    bit 3 ADC_AVG
    bit 2 ADC_AVG_INIT

Рекомендуемый простой режим постоянного мониторинга, 13 бит:

    bq_write8(0x2B, 0xA0)

Остановить ADC:

    bq_update_bits(0x2B, 0x80, 0x00)

Вариант one-shot, 13 бит:

    bq_write8(0x2B, 0xE0)

После one-shot MCU ждет REG0x21 bit 7 = 1. По завершении ADC_EN автоматически
очищается.


REG0x2C - выбор каналов ADC
-------------------------------------------------------------------------------

В этом регистре 1 означает отключение соответствующего канала:

    bit 7 IAC_ADC_DIS
    bit 6 IBAT_ADC_DIS
    bit 5 VAC_ADC_DIS
    bit 4 VBAT_ADC_DIS
    bit 2 TS_ADC_DIS
    bit 1 VFB_ADC_DIS

Для проекта нужны IAC, IBAT, VAC и VBAT. Значение по умолчанию уже оставляет эти
каналы включенными. Без необходимости REG0x2C лучше не перезаписывать.


REG0x2D/0x2E - измеренный ток IAC
-------------------------------------------------------------------------------

16 бит, little-endian, signed two's complement.

Для RAC_SNS = 5 мОм:

    IAC_mA = signed16(raw) * 0.8

Это ток на стороне USB-C-выхода.


REG0x2F/0x30 - измеренный ток IBAT
-------------------------------------------------------------------------------

16 бит, little-endian, signed two's complement.

Для RBAT_SNS = 5 мОм:

    IBAT_mA = signed16(raw) * 2

Это ток, потребляемый преобразователем от общей шины VBUS_C0.


REG0x31/0x32 - измеренное напряжение VAC
-------------------------------------------------------------------------------

16 бит, little-endian, unsigned.

    VAC_mV = raw * 2

Это фактическое выходное напряжение преобразователя.


REG0x33/0x34 - измеренное напряжение VBAT
-------------------------------------------------------------------------------

16 бит, little-endian, unsigned.

    VBAT_mV = raw * 2

В проекте это фактическое напряжение общей шины VBUS_C0 на стороне SRN.


REG0x3B и REG0x3C - скорость затворов и dead time
-------------------------------------------------------------------------------

По умолчанию установлены самые быстрые драйверы и dead time 45 нс. На этапе
первого запуска значения оставить по умолчанию. Изменять их только после проверки
осциллографом ringing, shoot-through и фронтов SW1/SW2.


REG0x3D - идентификация микросхемы
-------------------------------------------------------------------------------

Только чтение. Ожидаемое значение текущей ревизии datasheet: 0x12.

    bits [6:3] PART_NUM = 010b, BQ25756
    bits [2:0] DEV_REV  = 010b

При другом значении MCU не должен включать силовую часть, пока не проверено, что
установлена совместимая ревизия.


REG0x62 - входной ток reverse mode и динамическая компенсация
-------------------------------------------------------------------------------

    bits [7:6] IBAT_REV:
        00 = 20 А
        01 = 15 А
        10 = 10 А
        11 = 5 А

    bit 1 EN_CONV_FAST_TRANSIENT:
        0 = обычная компенсация
        1 = быстрая компенсация reverse mode

В проекте установлено около 200 мкФ на выходе. Перед EN_REV необходимо очистить
EN_CONV_FAST_TRANSIENT:

    bq_update_bits(0x62, 0x02, 0x00)

В reverse boost datasheet требует значительно больше емкости при значении 1.

IBAT_REV ограничивает ток, забираемый отдельным преобразователем от VBUS_C0.
Требуемый ток оценивается так:

    I_VBUS_C0_expected = Vout * Iout / (VBUS_C0 * efficiency)

Выбирается ближайший следующий больший предел 5/10/15/20 А. Нельзя выбирать
5 А для режима 5 В -> 5 В, 5 А: с учетом потерь входной ток будет больше 5 А.
Для такого режима нужен предел 10 А либо уменьшенный разрешенный выходной ток.

Пример установки 10 А и EN_CONV_FAST_TRANSIENT = 0:

    bq_update_bits(0x62, 0xC2, 0x80)


4. Условия, без которых EN_REV не сработает
================================================

BQ25756 включает reverse mode только если одновременно выполнено:

1. SRN выше 3 В.
2. DRV_SUP находится в допустимом диапазоне.
3. EN_REV записан в 1.
4. TS находится между BCOLD и BHOT, если EN_TS = 1.

Если REVERSE_STAT не стал равен 1, MCU должен прочитать REG0x22, REG0x23 и
REG0x24, а не повторять EN_REV бесконечно.


5. Полная последовательность включения одного BQ25756
========================================================

Шаг 1. Убедиться, что входная общая шина присутствует

    VBUS_C0 > 3 В
    внешнее +5V_DRV для DRV_SUP присутствует

Шаг 2. Проверить связь

    id = bq_read8(0x3D)
    ожидать id = 0x12

Шаг 3. Оставить reverse mode выключенным

    bq_update_bits(0x19, 0x01, 0x00)

Шаг 4. Отключить watchdog

    bq_update_bits(0x15, 0x30, 0x00)

Шаг 5. Запретить прямой заряд и HIZ

    bq_update_bits(0x17, 0x05, 0x00)

Шаг 6. Оставить аппаратную защиту ILIM_HIZ включенной

    bq_update_bits(0x18, 0x40, 0x40)

Шаг 7. Выбрать нормальный порог reverse undervoltage

    bq_update_bits(0x1E, 0x20, 0x00)

Шаг 8. Отключить fast transient compensation

    bq_update_bits(0x62, 0x02, 0x00)

Шаг 9. Установить IBAT_REV по доступной входной мощности

    bq_update_bits(0x62, 0xC0, выбранное_значение)

Шаг 10. Установить безопасный начальный ток выхода

    bq_write16_le(0x0A, значение_IAC_REV)

Шаг 11. Установить требуемое напряжение

    bq_write16_le(0x0C, значение_VAC_REV)

Шаг 12. Прочитать текущие ошибки

    fault = bq_read8(0x24)
    если fault & 0x7A != 0: не включать

Маска 0x7A проверяет VAC_OV, IBAT_OCP, VBAT_OV, TSHUT и DRV_OKZ. VAC_UV
на старте может отражать выключенный выход, поэтому его следует анализировать
отдельно в зависимости от состояния порта.

Шаг 13. Включить reverse mode

    bq_update_bits(0x19, 0x01, 0x01)

Шаг 14. Дождаться подтверждения

    ждать, пока bq_read8(0x23) & 0x04 != 0
    использовать конечный timeout

Шаг 15. Проверить фактическое напряжение

    VAC = bq_read16_le(0x31) * 2 мВ

Разрешать PD-контроллеру завершить переход питания только после того, как VAC
достигло целевого значения с выбранным программным допуском.

Шаг 16. Запустить ADC и периодический мониторинг

    bq_write8(0x2B, 0xA0)


6. Работа совместно с USB-PD-контроллером
================================================

Начальное состояние USB-C source-порта:

1. MCU знает общую доступную мощность, полученную через входной PD-порт.
2. MCU рассчитывает набор Source PDO, разрешенный для каждого выходного порта.
3. До переговоров выходной BQ подготавливается к 5 В.
4. PD-контроллер рекламирует разрешенные MCU Source PDO.
5. Потребитель выбирает PDO и отправляет Request/RDO.
6. PD-контроллер сообщает MCU выбранные напряжение и ток.
7. MCU сначала программирует IAC_REV, затем VAC_REV соответствующего BQ.
8. MCU проверяет REVERSE_STAT и измеренное VAC.
9. Только после достижения напряжения PD-контроллер продолжает переход к новому
   контракту и сообщает потребителю готовность источника.

Для постоянно 5-вольтового канала используется та же последовательность и тот же
I2C-драйвер. Отличается только постоянное значение VAC_REV = 0x03E8.

При изменении активного контракта:

1. Не записывать напряжение, превышающее выбранный потребителем PDO.
2. При уменьшении допустимого тока сначала уменьшить IAC_REV.
3. При изменении напряжения записать новый VAC_REV и контролировать VAC_ADC.
4. После стабилизации установить окончательный IAC_REV, если он был временно
   уменьшен.

Общий лимит мощности входного PD-контракта BQ25756 самостоятельно не знает.
Его обязан соблюдать MCU:

    сумма(Vout_i * Iout_limit_i / efficiency_i) <= доступная входная мощность

Необходимо оставить запас на питание MCU, PD-контроллеров, потери преобразования
и переходные процессы.


7. Периодический мониторинг
================================

Рекомендуемый период программного опроса: 10...50 мс под нагрузкой и 100...500 мс
в ожидании. Это проектные значения, а не требования datasheet.

На каждом цикле читать:

    REG0x23  REVERSE_STAT
    REG0x24  текущие аварии
    REG0x2D  IAC
    REG0x2F  IBAT
    REG0x31  VAC
    REG0x33  VBAT/VBUS_C0

Периодически для журналирования читать ClearOnRead-флаги:

    REG0x25
    REG0x26
    REG0x27

MCU также должен вычислять:

    Pout = VAC * IAC
    Pin_channel = VBAT * IBAT

Если сумма потребления каналов приближается к доступной входной мощности, MCU
должен уменьшить лимиты IAC_REV и обновить рекламируемые PD-профили.


8. Реакция на ошибки
======================

Потеря I2C или неверный PART_NUM:

    EN_REV не включать; отключить внешний силовой путь порта, если он имеется.

REVERSE_STAT неожиданно стал 0:

    прочитать REG0x24 и REG0x27;
    считать контракт питания нарушенным;
    уведомить PD-контроллер/отключить порт;
    не включать повторно до определения причины.

VAC_UV или REV_UVP:

    BQ25756 автоматически очищает EN_REV;
    проверить перегрузку, IAC_REV и входную мощность;
    после снятия нагрузки заново выполнить полную последовательность запуска.

VAC_OV:

    немедленно считать выход аварийным;
    очистить EN_REV;
    не возобновлять работу, пока измеренное VAC не станет безопасным.

IBAT_OCP:

    уменьшить разрешенную мощность порта;
    проверить выбранный IBAT_REV и общую входную шину.

TSHUT:

    очистить EN_REV;
    дождаться охлаждения;
    уменьшить ток или запретить профиль высокой мощности.

DRV_OKZ:

    очистить EN_REV;
    проверить внешний +5V_DRV;
    повторный запуск разрешать только после исчезновения REG0x24 bit 1.

TS Cold/Hot:

    BQ25756 очищает EN_REV;
    при текущем фиксированном делителе такое событие означает аппаратную ошибку
    цепи TS или REGN.


9. Последовательность штатного выключения
===========================================

1. Сообщить PD-контроллеру о прекращении/сбросе контракта, если порт подключен.
2. Уменьшить IAC_REV до минимально необходимого значения, если требуется плавное
   снижение нагрузки.
3. Очистить EN_REV:

       bq_update_bits(0x19, 0x01, 0x00)

4. Дождаться REVERSE_STAT = 0.
5. Проверить VAC_ADC.

Выходные конденсаторы могут сохранять заряд после EN_REV = 0. Если требуется
гарантированное быстрое снижение VBUS, необходим управляемый разрядный путь.
Не считать EN_REV самостоятельным физическим разрядником выхода.


10. Минимальный псевдокод
==========================

bool bq_start_reverse(bus, vout_mV, iout_mA, ibat_limit_bits)
{
    if (bq_read8(bus, 0x3D) != 0x12)
        return false;

    bq_update_bits(bus, 0x19, 0x01, 0x00); // EN_REV = 0
    bq_update_bits(bus, 0x15, 0x30, 0x00); // watchdog off
    bq_update_bits(bus, 0x17, 0x05, 0x00); // EN_HIZ=0, EN_CHG=0
    bq_update_bits(bus, 0x18, 0x40, 0x40); // keep ILIM_HIZ enabled
    bq_update_bits(bus, 0x1E, 0x20, 0x00); // REV_UV = 80% target
    bq_update_bits(bus, 0x62, 0xC2, ibat_limit_bits); // fast transient off

    uint16_t iac_rev = (iout_mA / 50) << 2;
    uint16_t vac_rev = (vout_mV / 20) << 2;

    bq_write16_le(bus, 0x0A, iac_rev);
    bq_write16_le(bus, 0x0C, vac_rev);

    uint8_t fault = bq_read8(bus, 0x24);
    if (fault & 0x7A)
        return false;

    bq_update_bits(bus, 0x19, 0x01, 0x01); // EN_REV = 1

    if (!wait_until_with_timeout(bq_read8(bus, 0x23) & 0x04))
        return false;

    bq_write8(bus, 0x2B, 0xA0); // continuous ADC, 13 bit
    return true;
}

В реальной реализации перед вычислениями обязательно ограничить:

    vout_mV: 5000...20000
    iout_mA: 400...5000

и округлять значения вниз к шагу 20 мВ / 50 мА, чтобы никогда не превысить
разрешенные PD-контрактом значения.

12. Ограничения этого документа
=================================

Этот файл описывает управление BQ25756 непосредственно от MCU. Конкретные
регистры TPS65987DDH, которыми MCU получает выбранный PDO/RDO и управляет
состоянием PD-порта, должны быть описаны отдельно: это другой I2C-протокол и
другой документ (SLVUBH2B). Нельзя смешивать адреса регистров PD-контроллера с
адресами BQ25756.
